## The lead magnet (weeks 4–10)
Format matters more than polish. **A compliance officer will not run `npm install`.** Two assets, two audiences:

1. **`.xlsx` model-inventory template** (ships in `public/FREE-AI-Model-Inventory-Template.xlsx`)
   → this is what *converts the buyer*. Spreadsheets travel through compliance departments; GitHub repos don't.
   Host it as a one-click download (no signup wall) on a simple landing page.
2. **The deployed dashboard + static checklist page** (`npm run build` → host `dist/` on Netlify/Vercel/Pages)
   → this is the "what good looks like" demo you screen-share. It impresses founders/CTOs.

The repo + README + a Loom = your business card for the *technical* audience.
The **xlsx** is the lead magnet for the *actual* buyer. CTA everywhere:
*"Free FREE-AI model-inventory template (.xlsx) → [link]."*

### Accuracy discipline (non-negotiable — you're "the FREE-AI person")
- Framework facts verified: **7 Sutras, 6 Pillars, 26 Recommendations, report dated 13 Aug 2025.**
- Checklist rows currently map to **Pillars and Sutras** (verified). They do **NOT** yet cite
  specific recommendation numbers — and you must **not invent them**. Before publishing post #1 or #2,
  open the RBI FREE-AI PDF and trace each row to its source paragraph/recommendation #.
  One wrong claim, screenshotted by a consultant, ends the credibility play.
